Music‑production/DJ tools audit for Apple M4 (March 2026)
Overview

You are building a local‑first DJ and music‑production platform on an Apple M4 Mac using Python 3.11, FastAPI and a React/Vite front‑end. This audit surveys the current landscape (late 2025 – early 2026) of audio analysis tools that could complement your pipeline: ingestion → audio analysis → stem separation → loop editing → export to Ableton/DAW. The report assumes you already installed Demucs‑MLX, allin1, madmom, librosa, soundfile/audioread, numpy/scipy/scikit‑learn, WaveSurfer.js and Essentia. Each section is grouped by task, with a summary table listing the tool name, install command, description, Apple Silicon (M4) compatibility, and comments on why it could be useful. Experimental or unstable projects are flagged. At the end, the top recommendations are highlighted.

Final remarks

The Apple‑Silicon MLX ecosystem now offers impressive performance for source separation and speech processing, but there are still no MLX ports for chord recognition, key detection, beat tracking or melody extraction. For those tasks, rely on CPU‑ or MPS‑enabled Python libraries such as BeatNet, skey and CREPE. Always check the licensing and maturity of each tool; research projects like cue‑detr and neural‑music‑fp may need adaptation before production use. Combining several tools—Demucs‑MLX, BeatNet, S‑KEY, pyCrossfade and CREPE—will give your platform state‑of‑the‑art analysis capabilities on Apple M4.

Here’s a high-level overview of the most important findings:

Stem separation: demucs-mlx remains the fastest and most accurate option on Apple M4, with a 6-stem model separating piano and guitar as well. A complementary option is mlx‑audio‑separator, which wraps several architectures (Roformer, MDX, Demucs) and offers even higher speedups over the original audio‑separator pipeline. Both run natively on the MLX stack.
Beat and downbeat tracking: BeatNet is the state‑of‑the‑art joint beat/downbeat/tempo tracker and supports streaming, real‑time and offline modes. It installs via pip install BeatNet and improves on madmom’s DBN approach using a CRNN plus particle filtering. For research‑grade experiments, the Beat Transformer is available but requires training.
Key and chord recognition: The self‑supervised S‑KEY model from Deezer offers modern key detection with CPU/MPS support; its Python API or CLI can detect keys in wav/mp3 files. For chords, the older Chordino‑based chord-extractor provides triad estimation, while pyace implements simple/deep ACE models.
Additional music features: audioFlux delivers a broad range of transforms, feature extractors and MIR modules (onset detection, beat tracking, pitch etc.) that can feed your loop‑quality or danceability scoring. For automatic mixing, mixingbear detects compatible mix points and performs crossfades; it can pick matching segments based on Camelot key and tempo.
Fingerprinting & duplicates: Tools like pyacoustid (Chromaprint bindings) and the research‑grade Neural Music FP package offer audio fingerprinting and duplicate detection, with the latter promising robustness to noise. Classic Dejavu remains a reliable Python implementation for local databases.

Overall, for immediate installation on your Apple M4 platform, I recommend starting with: demucs‑mlx, mlx‑audio‑separator, BeatNet, S‑KEY, and audioFlux. These tools collectively cover high‑quality stem separation, beat/downbeat detection, key detection, and rich audio feature extraction, all optimized for modern Apple hardware.
